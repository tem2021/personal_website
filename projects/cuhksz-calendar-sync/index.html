CUHKSZ deadlines → Google Calendar (Cursor automation)

This tutorial shows a simple workflow for CUHKSZ students: use Cursor to read Blackboard “What’s Due” items and create Google Calendar events automatically.

google_calendar_mcp cuhksz_bb_summary Google Calendar API video

What you will build

  1. A local script (bbinfo) exports Blackboard deadlines to a text file.
  2. A local MCP server (google_calendar_mcp) lets Cursor create Google Calendar events.
  3. A Cursor rule makes the command check bbinfo run the script and sync deadlines to your calendar.

Prerequisites

  • Cursor installed
  • Python 3.10+ on your machine
  • A Google account
  • Access to CUHKSZ Blackboard

Part 1 — Create Google Calendar API credentials

Follow the first half of this tutorial video to create an OAuth Client ID and download a client secret JSON file: https://www.youtube.com/watch?v=1sOhWoiJYZE

Goal: you should end up with a file like client_secret_....json on your computer. Do not commit this file.

Part 2 — Set up google_calendar_mcp in Cursor (MCP server)

Follow the README in google_calendar_mcp: https://github.com/tem2021/google_calendar_mcp

High-level steps:

  1. Install the MCP server in a Python environment (pyenv/venv both work).
  2. Configure Cursor to start the MCP server (recommended: create .cursor/mcp.json in your project).
  3. Set GOOGLE_OAUTH_CLIENT_SECRET_PATH to your downloaded client_secret_....json.
  4. Trigger any calendar tool once (for example: calendar_list) to complete OAuth authorization.

Tip: Create a separate calendar like “CUHKSZ Deadlines” so you can bulk-delete events if needed.

Part 3 — Export Blackboard deadlines with bbinfo

Use the script in cuhksz_bb_summary: https://github.com/tem2021/cuhksz_bb_summary

After you run it, it produces a text file (for example course_data_summary.txt) that contains:

  • Announcements
  • Current courses
  • Due items (the part we want)

Part 4 — Make it one command with a Cursor rule

The goal is: when you say check bbinfo, Cursor will:

  1. Run bbinfo to refresh the summary file.
  2. Parse the “Due items” section.
  3. Create (or update) Google Calendar events via MCP.

Suggested event format

  • Title: the due item title (keep it short)
  • Description: course name + “source: Blackboard”
  • Time: use the due datetime as event end time; set start to 30 minutes earlier
  • Timezone: make sure the datetime includes +08:00

How to avoid duplicates

Use a stable key (for example course + title + due_datetime). Before creating a new event, search existing events in the target calendar for the same key.

Troubleshooting

  • OAuth fails: delete the stored token file and authorize again (see google_calendar_mcp README).
  • No due items: confirm you can see “What’s Due” in Blackboard and the script can access it.
  • Wrong time: check timezone and the due datetime format in the exported file.

Security reminder: never commit your Blackboard credentials, OAuth client secret, or token files.
